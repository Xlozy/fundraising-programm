Passung zuordnen

Diese Stiftungen hatten eine alte, undifferenzierte Passung-Bewertung (1–5). Passung ist jetzt eine Beziehung zwischen Träger und Stiftung (D-039) — bitte den alten Wert pro Stiftung einem oder mehreren Trägern zuordnen, oder verwerfen, falls nicht mehr zutreffend.

Daten importieren

JSON-Array einfügen und mit «Vorschau» prüfen. Verweise auf Träger / Stiftung / Projekt werden per Name oder Kürzel gegen bestehende Datensätze abgeglichen — ohne Treffer wird die Zeile als Fehler markiert und beim Import übersprungen. Ein Treffer auf Name/Kürzel eines vorhandenen Datensatzes aktualisiert diesen, statt eine Dublette anzulegen. Bei Stiftungen können optional anforderungsprofil (ersetzt das ganze Profil) und kontakte (per Name abgeglichen) mitgegeben werden, bei Projekten optional budget.ausgaben/ budget.einnahmen (per Position/Quelle abgeglichen).

Ein gemeinsames GitHub-Repo als Datenspeicher (D-029) — wie ein Login: alle mit Zugriff auf Repo + Token arbeiten am selben Datensatz, keine Trennung nach Träger.
